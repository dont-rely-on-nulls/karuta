* About
This is a toy, general-purpose relational programming language. We are not held accountable for anything that

* Requirements

** Prolog
- [X] PL Parsing
- [X] PL Query Conjunction
- [X] Comments
- [X] PL Register Allocation
- [ ] PL Compilation
- [ ] PL Integer Literals
- [ ] PL List Literals
- [ ] Modules
- [ ] Special syntax for database support using ~@~
- [ ] Dedicated bytecode for database interaction
- [ ] Some sort of type checking (if provided a .krt file with the types)
     - If you provide a file with the types, use that
     - If not, you can pass a flag to skip the check
     - If another flag is provided, you impure the compiler to ask the relational engine
     - If nothing above, fuck you. I will fail.
